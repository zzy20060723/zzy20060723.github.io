Java Code Analysis!?!

启动题目可以看到有一个网址和一份源代码；

网址进入是一个登陆界面，登陆后有一个flag的PDF需要Admin权限查看。源代码是Java写的。同时题目中还给出了四个提示大意为关注’controllers’, ‘services’ and ‘security’这三个文件和jwt的两个属性和密钥在代码的位置。

直接把整个项目喂给AI可以知道默认生成一个普通用户user和一个Admin用户Admin他们的用户名和邮箱相同，密钥根据注释可知是一个固定的值1234；所以这道题的思路为使用user登录再升权到Admin。

首先登陆

按f12可以看到当前账户的jwt令牌

我们已知密钥为1234是可以直接在jwt解密/加密 - bejson在线工具修改我们的token

通过代码我们知道了默认Admin用户的用户名和邮箱，其中userld是账户权限Admin用户为2，user用户为1，其他为6；

修改之后我们就有足够的权限去查看flag的PDF

使用bp抓包修改后即可得到

More SQLi

SQLi注入是一种 代码注入攻击，攻击者通过在应用程序的输入字段中插入恶意 SQL 代码，从而篡改或执行未授权的数据库操作。这种漏洞通常源于应用程序未对用户输入进行严格过滤

题目已经说明是SQLiLite的注入

启动题目是一个登陆界面随便输入可以直接看到她的查询语句

有了查询语句可以构'or 1 and 1--即可绕过

还是一个注入，先查看表结构，看到了flag

从more_table中读取flag的内容

MatchTheRegex

启动题目后可以看到是一个输入框输入东西之后会出现弹窗

查看源码看到了这部分和输入框的数据处理有关这部分代码将我们的输入发送到/flag这个节点之后再见返回的内容显示在弹窗中

function send_request() {
let val = document.getElementById(“name”).value;
// ^p…..F!?
fetch(/flag?input=${val})
.then(res => res.text())
.then(res => {
const res_json = JSON.parse(res);
alert(res_json.flag)
return false;
})
return false;
}

这里有一个注释p…..F!?看起来像正则表达式的形式结合题目基本可以确定就是正则表达式，将他输入之后弹窗返回了flag。
