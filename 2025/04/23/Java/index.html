2.限制符

。Java 中有四种主要的访问限制符，分别是 public、private、protected 和默认（无修饰符），为了限定对数据访问权限，下面详细介绍它们的作用。

1. public（公共的）

  • 作用：public 是最宽松的访问限制符。使用 public 修饰的类、变量或方法可以在任何地方被访问，不管是在同一个包内，还是在不同的包中。
  • 使用场景：当你希望某个类或类的成员能够被广泛使用时，就可以使用 public 修饰符。例如，工具类中的公共方法、公共接口等。
  • 示例代码

2. private（私有的）

  • 作用：private 是最严格的访问限制符。使用 private 修饰的变量或方法只能在定义它们的类内部被访问，其他类无法直接访问。这有助于实现类的封装，保护类的内部数据不被外部随意修改。
  • 使用场景：常用于封装类的内部状态，通过提供公共的访问方法（如 getter 和 setter 方法）来间接访问和修改这些私有成员。
  • 示例代码

3. protected（受保护的）

  • 作用：使用 protected 修饰的变量或方法可以在同一个包内的任何类中被访问，同时也可以在不同包的子类中被访问。
  • 使用场景：当你希望某个类的成员在同一个包内可以被自由访问，并且在不同包的子类中也能被访问时，可以使用 protected 修饰符。

4. 默认（无修饰符）

  • 作用：如果类、变量或方法没有使用任何访问限制符修饰，那么它具有默认的访问权限，也称为包访问权限。具有默认访问权限的成员只能在同一个包内的类中被访问。

3.包

包类似于文件夹，在同一个包下的类中的public,protected的方法和属性可以访问；

1.声明

package 关键字来声明类所属的包；并不是Java文件处于同一个文件夹就是在同一个包内，必须由package来著名他的包。

2.导入

import关键字用于导入其他包；

常遇到的就像Scanner的使用，要导入 java.util.Sanner包；

当你使用其他包中的类创建对象时，另一个类中的main函数并不会对本类产生影响。

3.包的层级结构

包可以有层级结构，使用点号（.）来分隔不同层级的包名。例如，java.util是 java 包下的util子包，java.util.concurrent 是 java.util 包下的 concurrent 子包。这种层级结构类似于文件系统中的文件夹嵌套，便于进一步组织和管理代码。

4.系统包

Java 标准库提供了许多内置的包，例如：

  • java.lang：包含 Java 语言的核心类，如 Object、String、System 等，该包会被自动导入，无需使用 import 语句。
  • java.util：包含各种实用工具类和集合框架，如 ArrayList、HashMap 等。
  • java.io：包含用于输入输出操作的类，如 File、InputStream、OutputStream 等。

5.构造方法

1.定义

构造方法是一种特殊的方法，它在创建对象时被调用，主要用于对象的初始化工作。

2.特点

  2. 没有返回类型：构造方法没有返回类型，连 void 也不能有。

7.关键字

1.this

1. 引用当前对象的成员变量

当局部变量（如方法参数）与成员变量同名时，为了区分它们，可使用 this 关键字来引用当前对象的成员变量。

2. 调用当前对象的方法

在一个类的方法中，可以使用 this 关键字调用当前对象的其他方法。

在 calculate 方法中，this.add(x, y) 调用了当前对象的 add 方法。

3. 在构造方法中调用其他构造方法

在一个类中，可以使用 this() 语句在一个构造方法中调用另一个构造方法，但 this() 语句必须是构造方法中的第一条语句。

在无参构造方法 Student() 中，this("Unknown", 0); 调用了带参数的构造方法 Student(String name, int age)。

4. 返回当前对象

在方法中，可以使用 this 关键字返回当前对象，以便实现方法的链式调用。

在 append 方法中，return this; 返回当前对象，使得可以连续调用 append 方法。

5. 注意事项

  • this 关键字只能在实例方法、构造方法或实例初始化块中使用，不能在静态方法中使用，因为静态方法属于类，不依赖于对象的实例，而 this 代表的是当前对象。
  • 如果在一个方法中没有局部变量与成员变量同名，使用 this 关键字引用成员变量不是必需的，但使用它可以提高代码的可读性。

2.static

1. 静态变量（类变量）

  • 定义：静态变量属于类，而非类的某个实例。在类加载时，静态变量就会被分配内存空间，并且所有该类的实例共享同一个静态变量。
  • 语法：在变量声明前加上 static 关键字。

2. 静态方法（类方法）

  • 定义：静态方法属于类，而不是类的实例。可以直接通过类名调用静态方法，无需创建类的实例。
  • 语法：在方法声明前加上 static 关键字。

  • 定义：静态代码块是类中的一段代码，用 static 关键字修饰，在类加载时执行，并且只执行一次。常用于对静态变量进行初始化。

  • 静态方法只能访问静态成员：静态方法不能直接访问非静态成员（变量和方法），因为非静态成员属于对象的实例，需要通过对象来访问。
  • 非静态方法可以访问静态成员：非静态方法既可以访问非静态成员，也可以访问静态成员。
